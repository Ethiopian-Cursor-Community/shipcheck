ShipCheck
Hackathon Pitch • Cursor Agent SDK
Production Readiness Scanner

ShipCheck helps teams answer: “Can we ship this repo to production?”

One URL in. One clear report out: score, verdict, critical issues, quick wins, and an actionable fix plan.

Team: Behailu (Frontend) • Nahom (Backend) • Natenael (Cursor SDK)

The Problem

  • Developers spend too long deciding if a project is truly production-ready.
  • Important risks hide across tests, CI, secrets, dependencies, and docs.
  • Hackathon demos often show features, but not deployment readiness.

We wanted a tool that turns repo complexity into a clear go/no-go decision.

The Solution: ShipCheck

Input

Public GitHub repo URL

Output

Score + verdict + issues + quick wins + fix plan

  • Frontend dashboard with clean UX for scanning + report reading.
  • Backend cloning + orchestration with strict JSON output contract.
  • AI audit logic powered by Cursor Agent SDK.

Architecture

Frontend (React + Vite + Tailwind) → API (Express + TypeScript) → Clone repo (simple-git) → Cursor Agent SDK audit → structured report.

  • Shared TypeScript contract (`AuditReport`) across frontend + backend.
  • Safe temp clone + guaranteed cleanup.
  • Single endpoint: POST /api/audit.

How We Implemented Cursor SDK

  • Local runtime audit: Agent reads cloned repository directory.
  • Prompt enforces strict JSON output and production-readiness criteria.
  • Zod validation + parser fallbacks guarantee robust report parsing.
  • Verdict logic + repo summary fields improve business readability.

Runtime model: composer-2 (configurable)

Output safety: schema-validated JSON

UX Highlights

  • Step-by-step scan feedback: validating → cloning → analyzing.
  • Verdict badge + score ring for instant read.
  • Repo overview: what it does, type, language, frameworks, key files.
  • Copy JSON and quick reset for demo flow.

Design goal: non-engineers can understand results in seconds.

Value & Impact

For Teams

Faster release decisions and clearer priorities before shipping.

For Hackathons

Shows real AI usefulness beyond chat: concrete engineering outcomes.

  • Reduces “unknown unknowns” before launch.
  • Transforms AI output into concrete action plan.
  • Creates shared language between engineering and product.
